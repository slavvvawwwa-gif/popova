Эта версия заменяет дизайн-раздел из CLAUDE_CODE_BUILD_PLAN.md (v1).
Зафиксирована **только цветовая система**. Layout, типографские пары, motion и
конкретная композиция — на усмотрение Claude Code и скилла ui-ux-pro-max.

Сохраните как `design-system/MASTER.md`.

## 0. Контекст проекта

Персональный сайт-портфолио театрального режиссёра. RU/EN. Аудитория — театры,
продюсеры, фестивали, пресса. Тон — авторский, статусный, не корпоративный шаблон.

**Стек**: Next.js (App Router) + TypeScript + Tailwind CSS, i18n (RU основной,
EN с fallback на RU), Sanity.io как headless CMS, деплой на Vercel через GitHub.

## 1. Цветовая система (ЗАФИКСИРОВАНО — источник истины)

| Токен | HEX | Назначение |
|---|---|---|
| `--bg-primary` | `#0A0A0A` | Фон страницы |
| `--bg-surface` | `#1E1E1E` | Карточки, секции |
| `--text-primary` | `#EDEDED` | Основной текст, заголовки |
| `--text-secondary` | `rgba(237,237,237,0.4)` | Вторичный текст, подписи, неактивные пункты навигации |
| `--accent` | `#D6303C` | Единственный цветной акцент |

### Принцип использования `--accent` (`#D6303C`)
Это **сигнальный цвет, не декоративный**. Использовать точечно для:
- активного пункта навигации / текущего состояния;
- ссылок, hover-состояний, primary CTA;
- одного выделенного слова/фразы в заголовке или цитате;
- маленьких маркеров-меток (точка, линия, бордер) на выделенном элементе галереи/карточки.

**Не использовать** как заливку крупных областей, фон секций или массово в UI —
тогда эффект "одного укола цвета" пропадает.

### Accessibility-примечание
`#D6303C` на `#0A0A0A` имеет ограниченный контраст для мелкого текста — используй
акцент для крупного текста (≥18px), иконок, обводок, точек-маркеров и UI-элементов,
а не для мелкого body-текста. Основной читаемый текст — всегда `#EDEDED` на
`#0A0A0A`/`#1E1E1E` (высокий контраст).

## 2. Направление дизайна (вдохновение, НЕ строгая спецификация)

Свободно для интерпретации Claude Code / скилла `ui-ux-pro-max`, с учётом:

- Тёмная тема, почти монохромная, один цветной акцент (раздел 1).
- Editorial / bold arthouse — асимметричная композиция, крупная типографика,
  много "воздуха", фотографии/видео спектаклей как главный визуальный материал.
- Serif для заголовков (с характером — Fraunces/Playfair/Cormorant и похожие),
  нейтральный sans для текста — но финальный выбор шрифтовой пары можно доверить
  рекомендациям скилла.
- Можно использовать (на усмотрение): декоративные крупные элементы (цифры, метки),
  вертикальную/нестандартную навигацию, асимметричные сетки с разными aspect-ratio
  карточек — но не обязательно следовать буквально.

**Скилл `ui-ux-pro-max`**: используется для проработки layout, компонентов,
spacing, motion-паттернов (scroll-reveal, hover/lightbox, transitions),
accessibility-чеклиста и стек-специфичных рекомендаций. Палитра из раздела 1 —
приоритетна и заменяет генерируемую скиллом по умолчанию.

## 3. Карта сайта и языки

```
/ (главная)
/about — о режиссёре / биография
/works — портфолио (каталог спектаклей)
/works/[slug] — страница спектакля
/playbill — афиша
/press — пресса / отзывы / награды
/contacts — контакты
```
Переключатель RU/EN на каждой странице. Fallback на RU, если EN-перевод пуст.

---

## 4. Модель данных (для Sanity CMS)

### Performance (Спектакль)
`title_ru/en`, `theatre`, `year`, `role`, `genre/tags[]`, `short_description_ru/en`,
`full_description_ru/en` (rich text), `cover_image`, `gallery[]`, `video_links[]`
(YouTube/Vimeo), `status` (текущий/архив), `slug`.

### Playbill Entry (Афиша)
`performance_ref`, `date` (+время опц.), `venue/city`, `note`. Без ссылок на билеты.

### Press Item (Пресса/Награда)
`type` (рецензия/интервью/награда), `title_ru/en`, `source`, `date`,
`excerpt_ru/en`, `external_link`, `related_performance` (опц.).

### Bio (singleton)
`photo`, `bio_text_ru/en` (rich text), `timeline[]` (год + описание, опц.),
`cv_file_ru/en` (PDF).

### Contacts (singleton)
`email`, `social_links[]` (платформа + URL).

## 5. Функциональные требования (сводно)

- Каталог спектаклей (`/works`) с фильтром минимум по году; при 30+ записях —
  фильтр по жанру/тегам.
- Страница спектакля: галерея с lightbox, embed видео, связанная пресса.
- Афиша: дата + театр/город + название (ссылка на `/works/[slug]`), без билетных
  CTA; прошедшие даты — архивируются.
- Контакты: только ссылки (email, соцсети, мессенджер) — БЕЗ формы.
- CV — скачиваемый PDF (RU/EN), доступен с `/about` и/или `/contacts`.
- SEO: title/description на каждую страницу, Open Graph для превью.
- Accessibility: контраст ≥ 4.5:1 для основного текста, focus states,
  keyboard-доступный lightbox (Esc, focus trap), `prefers-reduced-motion`
  отключает все scroll/parallax/transition эффекты без исключений.
- Responsive: 375 / 768 / 1024 / 1440px — навигация и сетки адаптируются без
  потери асимметрии/иерархии (не "схлопывать" всё в одинаковые блоки).

---

## 6. Anti-patterns (явно запрещено)

- Generic SaaS-палитры, цветные градиенты, multi-color UI (см. раздел 1 — один акцент).
- Использование `--accent` как фона крупных областей.
- Одинаковые по размеру/aspect-ratio карточки во всех гридах без вариативности.
- Parallax 3+ слоёв или активный на мобильных устройствах.
- Эмодзи как иконки — только outline-иконки (Lucide/Tabler/Feather).
- Билетные CTA в афише.
- Форма обратной связи на `/contacts`.
